## Beta release

The initial beta of jiki — a lightweight, browser-based Node.js runtime.

### Core runtime

- Virtual filesystem (MemFS) with Node.js-compatible semantics, directory operations, symlinks, file watching, and snapshot serialization
- Kernel execution engine with module caching, code transformation, and the require/module system
- Shell with pipes (`|`), logical operators (`&&`, `||`), semicolons, and built-in commands (`ls`, `cat`, `cd`, `pwd`, `mkdir`, `rm`, `echo`, `export`)
- Package manager supporting npm and pnpm layouts, dependency resolution, semver handling, caching, and concurrent downloads
- esbuild-wasm transpiler for TypeScript/JSX with lazy WASM loading and bundling

### Dev server

- HTTP server running via Service Workers
- MIME type detection and cache control headers
- Hot Module Reload (HMR) support
- Virtual HTTP server routing via `ServerBridge`

### Framework support

- Next.js with Pages Router and App Router, file-based routing, API routes, and SSR
- React, Vue, Svelte, Astro, Nuxt, and Vite all supported via examples

### Node.js polyfills

Polyfills for `fs`, `path`, `process`, `http`, `crypto`, `stream`, `child_process`, `events`, `util`, `os`, and more.

### UI components (jiki-ui)

- CodeEditor with Shiki syntax highlighting, tab support, save shortcuts, and dirty state tracking
- Terminal with command input, history navigation, output display, and clear functionality
- BrowserWindow preview iframe with navigation controls and URL bar
- FileExplorer showing a hierarchical directory tree with selection highlighting

### Examples

24 working examples demonstrating React, Vue, Svelte, Next.js, Nuxt, Astro, Vite, Express, Hono, Jest, Vitest, Prettier, TypeScript, Node.js, Shell Scripting, Crypto, esbuild, and an Interactive Tutorial.
