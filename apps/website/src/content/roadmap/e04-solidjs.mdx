internalNotes: |
  Solid.js uses a custom JSX transform (solid-js/jsx-runtime) that compiles to fine-grained reactive DOM operations (not virtual DOM). The Vite dev server must handle Solid's Babel plugin or the newer Solid SWC plugin. Since jiki uses esbuild, need either: (a) a jiki plugin that runs babel-plugin-jsx-dom-expressions, or (b) use vite-plugin-solid via the Vite dev server. Example structure: a reactive counter with derived signals and effects. Demonstrates that jiki supports non-React JSX transforms.
